** Org-mode Task Syntax for AI Agents

*** Status format:
    STATUS: TODO | INBOX | BACKLOG | IN-PROGRESS | WAITING | DONE
    
    - TODO: Active work, ready to assign
    - INBOX: Frozen/not ready yet
    - BACKLOG: Low priority, maybe later
    - IN-PROGRESS: Currently working
    - WAITING: Done, needs review
    - DONE: Approved and closed
    
    PRIORITY: A (high) | B (medium) | C (low)
    Date: <YYYY-MM-DD> format (angle brackets)
    Tags: Colon-separated :bug:api:ui:

*** Common tags:
    :bug: :feature: :refactor: :docs: :test:
    :api: :ui: :db: :auth:
    Tags can mention components, types of work, libraries, etc.

*** Status transitions:
    - TODO → IN-PROGRESS: Starting complex task
    - TODO → WAITING: Simple task done, needs review
    - TODO → DONE: Simple task obviously complete
    - IN-PROGRESS → WAITING: Complex task done, needs review
    - WAITING → DONE: Human approved

*** Context for Agent section:
    - What agent needs to know
    - Related files/schemas
    - Constraints
    - Background info

*** Implementation Notes (for IN-PROGRESS):
    [YYYY-MM-DD] Agent: Progress update
    [YYYY-MM-DD] Agent: Files changed
    [YYYY-MM-DD] Agent: Questions for human

*** Review Notes (for WAITING):
    What human should verify
